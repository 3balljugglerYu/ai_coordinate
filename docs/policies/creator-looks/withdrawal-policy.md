# 撤回・削除ポリシー (P-4 ドラフト)

> **ステータス**: Stage 3 公開前ドラフト / 法務レビュー前

## 1. 撤回 (withdrawn) について

クリエイターは **いつでも** 自身の投稿を撤回できる。撤回されると投稿は
即座にホームカルーセル / 詳細ページから消える。

| 撤回前の状態 | 撤回後の挙動 |
|---|---|
| `draft` (= 未昇格) | 行 + Storage 画像を **即時完全削除** |
| `pending` (= 審査中) | `moderation_status='withdrawn'` に変更、行は残す |
| `visible` (= 公開中) | 同上、ホームから即消える |
| `removed` / `withdrawn` (= 既に終端状態) | 行 + Storage 画像を **完全削除** |

## 2. 過去の生成結果について

撤回した投稿で消費者が **既に生成 / ホーム投稿した画像** は、消費者の手元 (= 個人
ライブラリ / ホームの公開投稿) に残り続けます。これは撤回の対象外です。

理由:
- 生成結果画像は消費者の所有物 (= 消費者の生成クォータを消費して作られたもの)
- 消費者の同意なく削除すると消費者の作品ポートフォリオが破壊される
- 既に X 等で拡散された画像は技術的に回収不能

そのため、撤回しても **過去の生成結果には影響しません**。投稿時点でこの点を
理解した上で投稿してください (= 投稿フォーム同意項目 3 「衣装と背景の二次利用に
同意する」に対応)。

## 3. hidden_prompt の削除 (= 内部処理)

撤回後 **30 日経過** すると、紐づく内部生成プロンプト (= `user_style_template_secrets`)
を **自動で物理削除** します (= GDPR / 「忘れられる権利」対応)。

- 削除は毎日 03:30 JST に走る自動ジョブ (`creator_looks_secrets_cleanup_daily`) で実行
- 30 日は撤回ミスからの復旧余地として設定
- 削除後は同じ画像で再投稿すれば内部プロンプトは再生成されます

## 4. アカウント削除との関係

クリエイターがアカウント自体を削除した場合、
`auth.users` への外部キー (`submitted_by_user_id`) の `ON DELETE CASCADE` により、
紐づく `user_style_templates` 行も自動削除されます。
同様に Storage 画像 + secrets も連動して削除されます。

## 5. 復旧

撤回した投稿の復旧は原則として行いません。誤って撤回した場合は
同じ画像で新規投稿してください (= 投稿頻度上限 1 日 2 件にご注意)。

## 6. 法令上の権利

GDPR / 改正個人情報保護法に基づく「忘れられる権利」「アクセス権」「修正権」
の行使を希望される場合は、本サービスのお問い合わせ窓口からご連絡ください。
