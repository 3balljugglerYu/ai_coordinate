# モデレーションポリシー (P-3 ドラフト)

> **ステータス**: Stage 3 公開前ドラフト / 法務レビュー前

## 1. 審査体制

Creator Looks に投稿された画像はすべて、本サービス運営チームが
**人手で 1 件ずつ確認** する (= MVP 期は自動品質判定なし、計画書 ADR 参照)。

通常の審査時間: **24 時間以内** (運営営業日基準)

## 2. 審査基準

審査では以下の観点を確認する:

- 投稿規約 (P-1) への適合性
- 第三者の知的財産・肖像権の侵害がないか
- センシティブな表現の有無
- 投稿時の出所申告と実際の画像内容の整合性
- 投稿された画像の品質 (= meta-prompt 抽出が機能する程度の visibility)

## 3. 判定結果

| 判定 | 状態 (DB) | クリエイターへの通知 |
|---|---|---|
| 承認 | `moderation_status = 'visible'` | 通知 C (=「投稿が公開されました 🎉」) |
| 却下 | `moderation_status = 'removed'` | 通知 C (=「投稿について確認のお願い」+ 理由) |
| 公開後の非公開化 | 同上 | 同上 (= 公開後の規約違反発覚時) |

## 4. 通報フロー

(Stage 3 で本格実装予定)

- 消費者は投稿詳細ページから通報可能
- 通報の重み付き合計が閾値超過で自動 pending 化 (= 既存 post_reports と同じパターン)
- 通報内容を運営が確認 → 上記判定フローへ

Stage 1〜2 では通報機能は実装せず、運営が定期的に visible 投稿を巡回する形で対応する。

## 5. 緊急対応

著作権侵害・違法コンテンツ・人権侵害が疑われる場合は、
**24 時間以内** に該当投稿を非公開化し、運営内で精査する。
DMCA 申し立てへの対応は別途 `dmca-policy.md` 参照。

## 6. 監査ログ

審査結果はすべて `style_template_audit_logs` に記録される
(= submit / approve / reject / unpublish / withdraw / extract_failed)。
クリエイター本人は自分の投稿の audit log を閲覧可能 (Stage 2 以降の機能)。
